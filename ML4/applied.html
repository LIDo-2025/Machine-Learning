    Exploring PCA ML4

    Learning Objectives

    1 Explain PCA as a geometric projection onto directions of maximal variance
    2 Apply PCA to obtain one-dimensional and two-dimensional representations of data
    3 Interpret principal components in terms of variance captured and orientation in feature space
    4 Use explained variance ratios to assess information retained after dimensionality reduction
    5 Select an appropriate number of principal components using variance-based heuristics
    6 Apply PCA to high-dimensional biological data such as gene expression datasets
    7 Recognise the limitations of PCA, including its linearity and sensitivity to feature scaling

    Introduction

    In this section of the lesson, we'll apply Principal Component Analysis (PCA) to some examples. As a linear dimensionality reduction technique, PCA represents data in terms of new orthogonal directions which capture maximal variance. Rather than describing samples using the original feature axes, PCA re-expresses the data as projections onto these principal directions, allowing dominant patterns and structure to be identified, more clearly.

    By examining how variance is distributed across components, PCA provides both a compact representation of high-dimensional data and a quantitative basis for deciding how many dimensions are required to retain meaningful information.

    PCA terms

    The principal components (PC) in a dataset are obtained via the covariance matrix. Covariance is an extension of the variance metric, the measure of how much data points in a single feature vary across the population:

    $$\sigma^2 = \frac{1}{n} \sum_{i=1}^{n} (x_i - \mu)^2$$

    where $x$ is the data and $\mu$ is the mean of the data.

    From this equation, you can see that variance is simply the average of how far every value deviates from the population mean.

    Covariance extends this idea by taking the differences of one feature (x) and multiplying them by the differences of another feature (y):

    $$\text{Cov}(X, Y) = \frac{1}{n} \sum_{i=1}^{n} (x_i - \mu_x)(y_i - \mu_y)$$

    When we calculate the covariance of every feature against every other feature in a dataset, we produce a covariance matrix. On the diagonal of this matrix sit the variances of each feature (i.e. $(x_i - \mu)^2$), and off the diagonal are the covariances between pairs of features. The matrix is symmetrical about the diagonal.

    See the code below where we can recreate the covariance matrix used to create our 2-dimensional dummy dataset:

    The principal components (and their associated variances) are derived from the covariance matrix through matrix transformation algebra. Specifically, PCA finds the eigenvectors and eigenvalues of the covariance matrix. The eigenvectors are the directions (the PCs themselves), and the eigenvalues represent the amount of variance explained along each direction. By ranking the eigenvalues in descending order, PCA identifies which components capture the most variance across the entire dataset, allowing us to focus on the most informative directions and discard those that contribute little to overall variation.

    1-D descriptions with PCA

    With this knowledge lets use dimensionality reduction with PCA to find a single direction that captures maximal variance in our simple 2-dimensional data.

    We will be using Scikit-Learn's implementation of PCA. For more information on their implementation, see the documentation.

    To get the best 1-D representation of data, we instantiate the PCA class with one component (n_components = 1) to be returned, and transform the data to the feature space of that component.

    from sklearn.decomposition import PCA

# Initialize the PCA model
pcaResults = PCA(n_components=1) # specify no. of components

# Fit to data
pcaResults = pcaResults.fit(data)

# Transform the data, using our learned PCA representation
dim1_data = pcaResults.transform(data)

print( "Size of new dataset: ", dim1_data.shape )
    Size of new dataset:  (300, 1)

    From the fitted model 'pcaResults' we can now get:

    • The eigenvalue or scale of the variance for a PC from explained_variance_. This gives us the scale of the variance explained from this single vector.

    explained_variance = pcaResults.explained_variance_

# How much variance was captured compared to original data?
print(f"Variance for the first PC: {explained_variance}.")
    Variance for the first PC: [2.04849475].

    • Alternatively, you can find the fractional variance using explained_variance_ratio_. It tells us what fraction of the total variance is retained by the reduced 1-D description of the data.

    explained_variance_ratio = pcaResults.explained_variance_ratio_

# How much variance was captured compared to original data?
print(f"Fractional variance captured by first PC: {explained_variance_ratio}.")
    Fractional variance captured by first PC: [0.85762697].

    • The vector coordinates from components_, which give us the direction in space of the vector. The angle of this first PC can calculated from the arcus tangens atan2 of the vector. You can compare it directly to the angle in the above plot.

    vec1 = pcaResults.components_[0,:]

print(f"The vector as an array: {vec1}")

    2-D descriptions with PCA

    Our dataset has two features, therefore we can obtain two principal components. The number of informative PCs you can obtain from a dataset is the smaller of either:

    • The number of features

    • The number of samples

    As our dataset 300 samples $\times$ 2 features, the maximum number of PCs we can obtain is two. These two will be orthogonal to each other (form a right angle).

    # Initialize the PCA model
pcaResults = PCA(n_components=2) # specify no. of components

# Fit to data
pcaResults = pcaResults.fit(data)

# Transform the data, using our learned PCA representation
dim2_data = pcaResults.transform(data)

print( "Size of new dataset: ", dim2_data.shape )
print(" ")
print(pcaResults.components_)
print(' ')

vec1 = pcaResults.components_[0,:]
vec2 = pcaResults.components_[1,:]

    NOTE

    When reading about PCA you will come across the term Singular Value Decomposition (SVD). As mentioned in the scikit-learn documentation for PCA, when you apply PCA the algorithm in the background uses SVD to return the results. The difference between the two is technical but SVD is the more general method. Wikipedia has as mathematical introduction to PCA which also discusses the relationship to SVD.

    Gene expression data

    To bring it back to a real-world example, we will use PCA to find a few features that capture most of the variability in the gene expression dataset.

    import glob
from pathlib import Path
import csv

# Load all data
nGenes = 1000 # keep only these genes for now

filectr=0

datapath = Path("./data/childhood_data")    #/relative/path/to/folder/with/data

for txt in datapath.glob("*.txt"):
    b=read_csv(txt, sep='\t', usecols=[1])
    b=reshape(b.values, (1,54675)) # each file is 1 observation sample i.e. 1 row.
    if filectr==0:
        coln =  read_csv(txt, sep='\t', usecols=[0])
        a = b
    else:
        a=append( a, b, axis=0)
    filectr+=1

# convert 2D array to DataFrame
geneData = DataFrame( data=a[:,range(nGenes)]- mean(a[:,range(nGenes)], axis=0), columns = coln.values[range(nGenes)])

    So far we have been working with dummy data where values are on the same scale, that is, the values across each feature are relatively proportional to one another. However, when working with real-world data this is often not the case. Several features will span different scales, with some below 1 and others above 1000.

    As we have seen, PCA finds the components that explain the most variance by ranking the magnitude of each eigenvector (specifically, by the associated eigenvalue, which represents the scale of variance). From the PCA terms section, we know that variance is the average squared difference from the mean. Features with larger scales will dominate the PCA space as their absolute difference to the mean will be larger, even if their normalised variance isn't actually that large. This means PCA will prioritise directions aligned with high-magnitude features, which may not reflect the true underlying patterns in your data.

    To counter this, you should always rescale your data to have standard mean and variance using a tool such as StandardScaler(), which we have encountered previously. A scaler will typically rescale the data to have a mean of 0 and unit variance of 1. This makes features more comparable whilst retaining their variance.

    $$z = \frac{x_i - \mu}{\text{stdev}(x)}$$

    where $\mu$ is the feature mean and $\text{stdev}(x)$ is the standard deviation of the feature.

    from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()

geneData_scaled = scaler.fit_transform(geneData)

    Calling PCA without specifying the number of components will generate a PCA where the number of components equals the smaller of either the number of features or the number of samples. Since we have a short and wide dataset, this means our maximum number of components is 80. Attempting to extract more components than unique sample points simply results in generating superfluous, over-fitted noise rather than meaningful signals.

    GenePCA = PCA()
GenePCA = GenePCA.fit(geneData_scaled)

nComp = len(GenePCA.components_)

print(f"Number of components: {nComp}")

    Of the 80, how many features should we retain? To investigate this question, we plot the total (cumulative) variance for retaining the top k modes. The more we use, the higher the cumulative variance.

    A common heuristic for choosing the number of components is by defining a set threshold for the total explained variance.

    Thresholds commonly vary between 0.8-0.9, depending on the structure of the PCs, e.g. depending on whether there are a few top PCs or many small PCs, but also depending on the expected noise in data, and on the desirable accuracy of the reduced dataset. While a dimensionality reduction is convenient, it always loses some information.

    As an example, we can check how many PCs we need to retain 95 % of the variance.

    cum_bool = cumExpVar >= threshold

print(cum_bool)
print(" ")

# index of first PC whose cumulative variance reaches threshold
n_components = argmax(cum_bool) + 1  # +1 because of zero based indexing

n_components

print(
    f'Number of features needed to explain {threshold} fraction of total variance is {n_components}.'
)

    There are many alternative methods for estimating the number of features. These include:

    • Plot explained variance of individual PCs and cut off when there is a sharp drop/saturation in the values.

    • Cross-validation - Find number of PCs that maximize explained variance on held-out data (using a bi-cross validation scheme).

    • Visual inspection or interpretable PCs

    Feature Contributions

    When reducing a dataset you are compressing the information it contains. This may improve your machine learning model's performance or your ability to visualise clusters in high-dimensional space. However, you are then left wondering what true biological factors are represented by the PCs that now best explain your data. Unlike some machine learning methods, PCA is not a black box. It is grounded in strong mathematical theory, which means we can extract the contribution (or loadings) that each feature makes towards each PC. In fact, these loadings are stored in the components_ array, which contains the PC vector coordinates.

    Each value in components_ represents the loading (contribution) from each feature to a given PC. The first feature in your matrix is at index 0, the second at index 1, and so on. Let us use this to find which feature from the Gene dataset contributes most to the first PC.

    Beyond dimensionality reduction

    After reducing the dataset from hundreds or thousands of genes to fewer features (PCs), it can be used:

    • To find groups of genes that seem to be co-regulated (using clustering).

    • To build classifiers to predict the likelihood of juvenile or late-onset rheumatoid disease. For example, one feature can be highly variable in the sampled population and therefore has high explained variance, but it may not be predictive for the current objective of predicting disease phenotype (e.g. eye pigmentation genes).

    • To find what combination of genes the top features correspond to.

    • To suggest what individual genes are most variable.

    • To discover co-expression patterns of multiple genes.

    When does PCA fail?

    Although PCA is the first and simplest method of exploring high-dimensional datasets, there are important caveats to keep in mind:

    1. PCA is a linear method
    Nonlinear patterns of co-expression will be seemingly broken up into many features. Thus, the ability to find whether genes are co-regulated may be reduced if the effects are nonlinear.
    1. PCA depends on scaling of individual measurements.
    As PCA measures variance along different directions, changing the scale of one variable (for example from cm to mm, or fold-change to actual levels) may drastically change the dominant "features" extracted. You must therefore try to select a scaling method that works best to preserve the underlying characteristics of your data. Typically, there will be an domain standard method to deal with re-scaling data, so be sure to investigate.
    1. Estimating number of features is heuristic, and depends on the purpose.
    The definition of what is the 'relevant' number of features can depend on:
    • the signal-to-noise ratio in data. For example, noisier or smaller datasets make it harder to accurately estimate smaller PCs. In such cases, a more conservative threshold should be used.

    • the classifier or regression performance following the dimensionality reduction.

    Summary

    We introduced and looked at dimensionality reduction as a practical and conceptual response to the challenges posed by high-dimensional data. We observed how data can be re-expressed as projections onto orthogonal directions that capture maximal variance. By examining variance along different directions, PCA provides a principled way to identify dominant structure, reduce redundancy between correlated features, and construct lower-dimensional representations of complex datasets. The lessons demonstrate how explained variance can be used to guide the choice of dimensionality, while also highlighting the assumptions and limitations of PCA, including its linear nature and sensitivity to feature scaling. Together, these ideas establish PCA as a foundational tool for exploring, visualising, and simplifying high-dimensional biological data.
